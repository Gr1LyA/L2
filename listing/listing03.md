Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
<nil>
false

*os.PathError - тип данных, реализующий интерфейс error.
Из функции возвращается объект != nil. Равен nil только поинтер указывающий на само значение внутри интерфейса

	Интерфейс под капотом хранит указатель на структуру-таблицу, которая хранит некоторые 
метаданные о типе и список методов, используемых для удовлетворения интерфейса, а также
указатель на фактическую переменную с конкретным типом.

	Пустой интерфейс хранит в себе описание типа и поинтер на значение
	не харнит указателя на таблицу
Базовое представление пустого интерфейса задокументировано в пакете reflect: 
type emptyInterface struct {
   typ  *rtype            // слово 1 с описанием типа
   word unsafe.Pointer    // слово 2 со значением
}
